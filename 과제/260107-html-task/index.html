과제 1: robots.txt 조사 보고서

1. robots.txt의 정의

웹사이트의 루트 디렉토리에 저장되는 텍스트 파일로, 웹 크롤러와 같은 로봇들이 사이트의 특정 부분에 접근할 수 있는지 여부를 지시하는 국제 규약이다.

2. 주요 역할

  • 검색 엔진 최적화(SEO)를 위해 불필요한 페이지의 인덱싱을 방지한다.
  • 서버 자원을 소모하는 무분별한 크롤링을 제한하여 서버 부하를 관리한다.
  • 관리자 페이지나 민감한 데이터가 포함된 경로에 로봇이 접근하지 못하도록 설정한다.

3. 사용 예시

# 예시 1: 모든 로봇의 관리자 폴더 접근 금지
User-agent: *
Disallow: /admin/

# 예시 2: 구글 로봇만 전체 접근 허용
User-agent: Googlebot
Allow: /

4. 활용 권장 상황

사이트 내에 중복된 콘텐츠가 많아 검색 순위에 악영향을 줄 우려가 있거나, 회원 전용 페이지처럼 검색 결과에 노출되어서는 안 되는 경로가 있을 때 사용한다.

과제 3: UI 컴포넌트 라이브러리 분석

1. MUI (Material-UI)

특징: 구글의 Material Design 시스템을 따르며 컴포넌트의 완성도가 매우 높습니다. 리액트 환경에서 가장 널리 쓰인다.

체계적이고 규격화된 기업형 프로젝트에 적합하다.

2. Bootstrap

특징: 전 세계에서 가장 유명한 프레임워크로, HTML 클래스만으로 반응형 레이아웃을 빠르게 구축할 수 있다.

문서화가 잘 되어 있어 초보자가 사용하기에 가장 친절한 도구이다.

3. Ant Design

특징: 대규모 데이터 처리와 복잡한 입력 폼이 필요한 관리자용 웹사이트(Admin) 제작에 특화되어 있다.

차트나 테이블 등 업무용 위젯이 풍부하여 생산성이 높아 보인다.

4. Chakra UI

특징: 개발자가 스타일을 자유롭게 조합할 수 있는 유연성을 제공하며, 웹 접근성 준수가 뛰어납니다.

인상: 모던한 감각의 개인 프로젝트나 스타트업 서비스에 어울리는 유연함이 매력적입니다.

종합 의견

가장 마음에 든 라이브러리: Chakra UI

이유: 학습 곡선이 낮으면서도 디자인이 세련되어, 현재 개인적으로 계획 중인 프로젝트에 적용했을 때 가장 빠르고 깔끔한 결과물을 낼 수 있을 것 같다.
